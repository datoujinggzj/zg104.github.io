What is Linear Regression?

Suppose we have the data
D=(x1,y1),⋯ ,(xN,yN)D = (x_1,y_1), \cdots , (x_N,y_N)D=(x1​,y1​),⋯,(xN​,yN​)
where
X=(x1,x2,⋯ ,xN)T,Y=(y1,y2,⋯ ,yN)TX = (x_1,x_2,\cdots,x_N)^T,Y = (y_1,y_2,\cdots, y_N)^TX=(x1​,x2​,⋯,xN​)T,Y=(y1​,y2​,⋯,yN​)T
Suppose
h(x)=hθ(x)=θ0+θ1x1+θ2x2+⋯+θNxN=θTXh(x) = h_\theta(x) = \theta_0+\theta_1x_1+\theta_2x_2+\cdots+\theta_Nx_N = \theta^TXh(x)=hθ​(x)=θ0​+θ1​x1​+θ2​x2​+⋯+θN​xN​=θTX

Linear regression assumes that the features and responses meet a linear relationship. In fact, the ability to express linear relationships is very powerful. The influence of each feature on the result can be reflected by the previous parameters, and each feature variable can be mapped to a function first, and then participate in linear calculations. In this way, the nonlinear relationship between features and results can be expressed.

x1,⋯ ,xNx_1, \cdots,x_Nx1​,⋯,xN​represents different features, θ\thetaθ is parameter here, which is also the weights, representing how much we should weight the features to regress on the target. Then, we need to find some method to evaluate θ\thetaθ. J(θ)J(\theta)J(θ) is defined to describe how well the model performs.

Gradient Descent

We are looking for θ\thetaθ which makes J(θ)J(\theta)J(θ) minimized, so the question is summarized as follow：

  1. First assign random values ​​to the parameters
  2. Changing the parameters makes the loss function decrease in the direction of gradient descent. The gradient direction is determined by the partial derivative. Because the value is the minimum value, the gradient direction is the opposite direction of the partial derivative. There are two iteration methods:
      • Batch gradient descent:
        Update the parameters after obtaining the error of all the training data
      • Incremental gradient descent:
        The parameters are updated at each scan step (may be constantly converging and hovering)
  3. Repeat until convergence

Logistic Regression

What is LR?

Summarize in one sentence:
Logistic regression assumes that the data obeys the Bernoulli distribution, through the method of MLE, using the gradient descent to solve the parameters, so as to achieve the purpose of binary classification.

First of all, although logistic regression is called regression, it is not a regression method. It is mainly used to solve binary classification problems. In the face of classification problems, the linear regression above seems powerless. As you can see from the figure below, linear regression is acting as discrete When the variable y exists, it can only be fitted with a straight line, and the predicted y may exceed the range of 0 to 1, and is very sensitive to outliers.

Then, we want to introduce non-linear elements to solve the questions above, we expect p(y=1∣X;θ)p(y=1|X;\theta)p(y=1∣X;θ) between 000 and 111, and the result not to be sensitive to the outliers.

For a binary classification, we suppose

Naive Bayes

Discriminant Model vs. Generative Model

Discriminant Model

We know that in fact, machine learning is to find a way to predict the target yyy from the feature XXX, that is, to find the probability p(y∣X)p(y|X)p(y∣X).

Then for the discriminant model, we only consider estimating the conditional probability, which is to establish the discriminant function under the condition of a limited sample, without considering the generation model of the sample, and directly study the prediction model, which can also be understood as in the binary classification problem, p(y=1∣X)+p(y=0∣X)=1p(y =1|X) + p(y=0|X) = 1p(y=1∣X)+p(y=0∣X)=1, the sum of conditional probabilities is equal to 111.

Generative Model

The generative model obtains p(y,X)p(y,X)p(y,X), we need to find the joint distribution probability between XXX and the unused target, and then win a big win. Simply put, find two joint probability distributions, and compare to select the bigger one.

What is Naive Bayes

First of all, Naive Bayes is a generative model. The Naive Bayes classifier is a series of assuming strong independence between features using Bayes theorem-based simple probabilistic classifier.

Naive Bayes is a simple technique for constructing classifiers: models that assign class labels to problem instances, represented as vectors of feature values, where the class labels are drawn from some finite set. There is not a single algorithm for training such classifiers, but a family of algorithms based on a common principle: all naive Bayes classifiers assume that the value of a particular feature is independent of the value of any other feature, given the class variable. For example, a fruit may be considered to be an apple if it is red, round, and about 10 cm in diameter. A naive Bayes classifier considers each of these features to contribute independently to the probability that this fruit is an apple, regardless of any possible correlations between the color, roundness, and diameter features.

For some types of probability models, naive Bayes classifiers can be trained very efficiently in a supervised learning setting. In many practical applications, parameter estimation for naive Bayes models uses the method of maximum likelihood; in other words, one can work with the naive Bayes model without accepting Bayesian probability or using any Bayesian methods.

Despite their naive design and apparently oversimplified assumptions, naive Bayes classifiers have worked quite well in many complex real-world situations. In 2004, an analysis of the Bayesian classification problem showed that there are sound theoretical reasons for the apparently implausible efficacy of naive Bayes classifiers.[6] Still, a comprehensive comparison with other classification algorithms in 2006 showed that Bayes classification is outperformed by other approaches, such as boosted trees or random forests.[7]

An advantage of naive Bayes is that it only requires a small number of training data to estimate the parameters necessary for classification.

Core idea: Bayesian formula & Independence assumption

Core (2): Independent Assumption

From now on, the “naive” conditional independence assumption has come into play.

What is conditional independence?

When two events AAA and BBB occur in another given event YYY, the occurrence of AAA and the occurrence of BBB are independent in terms of the conditional probability distribution, that is, $ A$ and BBB are conditionally independent when a given YYY occurs. If and only if Y is known to occur, knowing whether AAA has occurred or not does not help to know whether BBB has occurred or not, and knowing $ Whether B$ occurs or not does not help to know whether AAA occurs or not.

So we assume that each feature XiX_iXi​ is conditionally independent to other features XjX_jXj​, which means
p(Xi∣y,Xj)=p(Xi∣y)p(X_i|y,X_j) = p(X_i|y)p(Xi​∣y,Xj​)=p(Xi​∣y)

For i≠ji \neq ji​=j, the joint distribution model can be expressed as

Part 1: Gaussian Naive Bayes（GBN）

If you are dealing with continuous data, a common assumption is that these continuous values ​​are Gaussian distribution. For example, suppose there is a continuous feature xxx in the training set. We first classify the data according to categories, and then calculate the mean and variance of xxx in each category.

We have

p(y=k∣X=x)=πkfk(x)∑l=1kπlfl(x)p(y=k|X=x)=\frac{\pi_kf_k(x)}{\sum_{l=1}^{k}\pi_lf_l(x)}p(y=k∣X=x)=∑l=1k​πl​fl​(x)πk​fk​(x)​

Where πk\pi_kπk​ is the prior probability, and fk(x)f_k(x)fk​(x) is the likelihood function.

Let’s use a feature x=vx=vx=v as an example

p(x=v∣y=k)=fk(x)=12πσy2exp(−(v−μy)22σy2)p(x = v|y=k) =f_k(x)= \frac{1}{\sqrt{2\pi\sigma_y^2}}exp\Big(-\frac{(v-\mu_y)^2}{2\sigma_y^2}\Big)p(x=v∣y=k)=fk​(x)=2πσy2​​1​exp(−2σy2​(v−μy​)2​)

Another commonly used technique for dealing with continuous numerical problems is by discretizing continuous numerical values. Generally, when the number of training samples is small or the exact distribution is known, the probability distribution method is a better choice. In the case of a large number of samples, the discretization method performs better, because a large number of samples can learn the distribution of the data. Since Naive Bayes is a typical method that uses font color=red>a large number of samples (The larger the amount of calculation, the higher the classification accuracy), so Naive Bayes methods use discretization methods instead of probability distribution estimation methods.

The following is the working principle of the GBN classifier. At each data point, at this point and each class-mean z-score can be calculated.

If a given class and feature value did not appear together in the training set, then the probability will be 000 under the frequency-based estimation. This will be a problem. Because when multiplying with other probabilities, all other probabilities will be removed. Therefore, it is often required to modify the probability estimation of each small sample to ensure that there will be no probability of 000.

Although the independence assumption is often inaccurate in practice, several features of the naive Bayes classifier make it possible to obtain surprising results in practice. In particular, the decoupling (decoupling in mathematics refers to turning a mathematical equation containing multiple variables into a set of equations that can be represented by a single variable, that is, variables no longer directly affect one at the same time The result of the equation, thereby simplifying analysis and calculation) between various conditional features means that the distribution of each feature can be independently estimated as a one-dimensional distribution.

This alleviates the obstacles caused by the curse of dimensionality. When the number of features of the sample increases, the sample size does not need to be doubled. However, in most cases, Naive Bayes cannot make a very accurate estimate of the class probability, but this is not necessary in many applications.

For example, in the naive Bayes classifier, the correct classification can be obtained according to the maximum posterior probability decision rule as long as the posterior probability of the correct class is higher than other classes. Therefore, no matter whether the probability estimate is slightly or even severely imprecise, it does not affect the correct classification result. In this way, the classifier can be robust enough to ignore the flaws in the naive Bayes probability model.

Core Idea of LDA

As a dimensionality reduction and classification technology of supervised learning, LDA can project data to a low-dimensional space under the premise of maintaining classification to reduce computational complexity.

LDA requires that the variance between classes is the largest and the variance within the classes is the smallest to ensure that the data distance between different categories is as large as possible in the same classification data set after projection.

Summary

The basic idea of ​​the naive Bayes algorithm is to establish a joint probability distribution p(X,y)p(X,y)p(X,y) between the feature XXX and the output yyy. When predicting a given feature, use Bayes Theorem Find all possible outputs p(X|y), and take the largest one as the prediction result. The advantage is that the model is simple, efficient, and widely used in many fields. Naive Bayes and LDA/QDA are all generative models, and the convergence speed will be faster than the discriminative model (such as logistic regression).
